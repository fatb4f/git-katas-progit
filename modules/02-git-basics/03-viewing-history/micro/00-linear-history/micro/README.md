# Micro-kata — Linear History

## Objective

Use the prepared `linear-5` repository fixture to identify boundary commits and newest commit provenance.

## Command family

```bash
git log
```

## Required outputs

Write:

- newest commit hash to `.answers/newest_commit`
- oldest commit hash to `.answers/oldest_commit`
- newest commit author field to `.answers/author_field`
- newest commit subject to `.answers/commit_subject`

## Learner constraint

This is a read-only analysis kata. Do not mutate refs, commits, index, or worktree state while collecting answers.
